# Security Review Report

**Version:** 2.1.2
**Date:** 2026-06-13

---

## Scope

- License System
- Update System
- Installers
- File Parsing
- Exports

## Findings

### Finding 1: Demo-Grade HMAC License Key — INFORMATIONAL
- **File:** `license.go:15`
- **Details:** `const DemoSecret = "asf-enterprise-secret-2024"` is a hardcoded HMAC secret
- **Risk:** Low (intentional, documented)
- **Mitigation:** Code comment explicitly states "Demonstration-only HMAC key. It provides obfuscation, NOT security." Ed25519 signing path (`license_ed25519.go`) is available for production use. This is a deliberate trade-off for the open-source community edition.

### Finding 2: Unsafe Temp Directory Usage — LOW
- **File:** `paths.go:12`, `paths.go:27`
- **Details:** Uses `os.TempDir()` for config and cache directories
- **Risk:** Low — temporary files are config/cache files within a scoped subdirectory (`asf/config`, `asf/cache`). No sensitive data written. No world-readable permissions on files with sensitive content (license file uses `0600`).

### Finding 3: No Path Traversal in File Operations — CLEAN
- All `filepath.Join` calls use either:
  - Predefined base directories with file names from `filepath.Base()` (directory traversal stripped)
  - Internal IDs (UUIDs/hashes generated by system)
  - Fixed output file names
- `portfolio_cli.go:96` uses `filepath.Base(resultPath)` which strips directory components

### Finding 4: Safe File Permissions — CLEAN
- License files: `0600` (owner-only read/write)
- Portfolio records: `0644`
- Export files: `0644`
- Config files: `0644`
- All files use appropriate restrictive permissions with no world-writable bits

### Finding 5: No Unsafe Symlink Handling — CLEAN
- Only symlink usage: `doctor.go:13` uses `filepath.EvalSymlinks` for safe resolution

### Finding 6: No Unsafe Archive Extraction — N/A
- No archive extraction code present in the codebase

### Finding 7: File Parsing — CLEAN
- YAML parsing uses `gopkg.in/yaml.v3` (well-maintained)
- JSON parsing uses standard `encoding/json`
- PDF parsing uses `github.com/ledongthuc/pdf` (read-only, no execution)
- File parser (`parser.go`) handles Draw.io/MxGraph XML parsing with no external entity expansion concerns

## Recommendations

1. **License System:** Document that production deployments should replace `DemoSecret` or use Ed25519 licenses. The existing Ed25519 path (`license_ed25519.go`) is the recommended upgrade path.
2. **No critical, high, or medium severity issues found.**

## Conclusion

**Overall: LOW RISK — No exploitable vulnerabilities found.**
